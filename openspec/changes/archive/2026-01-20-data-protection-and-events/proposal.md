# Change: 对象版本控制、跨区复制、事件通知与访问日志

## Why
- 对齐 S3 兼容矩阵，补齐数据保护与审计能力，满足用户对版本管理、合规和安全追踪的诉求。
- 支持跨区域灾备与数据分发，满足业务连续性和就近访问需求。
- 提供事件驱动能力与访问日志，便于构建自动化流水线、审计与故障分析。
- 作为客户端 SDK，需要暴露完整配置接口与类型，减少用户直接操作底层 HTTP 的复杂度。

## What Changes
- 桶版本控制：新增版本控制开关的设置/查询接口，扩展对象操作支持 `versionId`（Get/Head/Delete/Copy），提供对象版本/删除标记的列表接口。
- 跨区复制：新增复制配置的设置/获取/删除接口，支持多规则、前缀/标签过滤、目标区域/桶/存储类型定义，并在相关请求中透传复制所需元数据。
- 事件通知：新增桶事件通知配置的设置/获取/删除接口，支持常见事件类型（对象创建/删除/还原/标签变更等）与多种目标（队列/主题/回调），提供类型与校验。
- 访问日志：新增桶访问日志配置的设置/获取/删除接口，支持目标桶/前缀/授权配置，并暴露相应的序列化与错误处理。
- 辅助更新：新增/扩展选项与错误类型，增加示例与 README/CHANGELOG 说明，完善核心测试覆盖。

## Impact
- 影响的规范：core-api（桶配置、对象操作、错误与类型）。
- 影响的代码：bucket 配置模块（versioning/replication/notification/logging）、object 模块（versionId 支持与版本列表）、types/options 定义、客户端入口与校验逻辑、测试与示例、文档与变更记录。
