# Change: 迁移 S3 示例到新 API

## Why
旧工程 `old/examples/s3/` 目录包含 71 个完整的 S3 API 使用示例，这些示例演示了各种存储桶和对象操作。为了帮助用户从旧 API 迁移到新的模块化 API，需要将这些示例重构并迁移到新的代码结构中，同时避免版权纠纷。

当前问题：
- 旧示例使用 MinIO SDK 的 API 风格（`minio-go/v7`）
- 新 SDK 采用了服务化和选项模式的 API 设计
- 缺少覆盖所有功能的完整示例集
- 旧示例代码包含 MinIO 版权声明

## What Changes
- 将 71 个 S3 示例迁移到 `examples/s3/` 目录
- 重构代码以适配新的 API 设计（服务化 + 选项模式）
- 移除 MinIO 版权声明，添加 RustFS 项目声明
- 重新组织代码结构，改进命名和注释
- 确保示例代码清晰、易读且符合 Go 最佳实践
- 保持功能完整性的同时进行独立实现

**示例分类：**
- 存储桶操作（创建、删除、列表、存在性检查等）
- 对象操作（上传、下载、复制、删除等）
- 高级特性（加密、版本控制、标签、生命周期等）
- 预签名 URL（GET、PUT、POST Policy）
- 流式上传和下载
- 多部分上传
- 对象锁定和保留

## Impact
- 影响的规范：examples（新增）
- 影响的代码：
  - 新增 `examples/s3/` 目录及其下所有示例文件
  - 可能需要更新 `README.md` 和 `README.zh.md` 中的示例引用
  - 不影响核心库代码（`bucket/`、`object/`、`client.go` 等）
